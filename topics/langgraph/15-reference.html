Core Graph APIs
StateGraph · compile · invoke · stream · state inspection
▾
StateGraph(schema)
StateGraph(State: type, config_schema=None)
Creates a new graph builder. State is your TypedDict, dataclass, or Pydantic model. config_schema allows typed config dicts for runtime configuration.
First call in every graph State schema is immutable after compile()
compile()
builder.compile(checkpointer=None, interrupt_before=[], interrupt_after=[], debug=False)
Validates the graph topology (checks for unreachable nodes, missing edges) and produces a CompiledGraph. Must be called before any execution. Re-compile whenever you change the graph structure.
After all add_node/add_edge calls Raises ValueError for unreachable nodes
invoke()
graph.invoke(input: dict, config: dict = None) -> dict
Runs the graph synchronously to completion. Returns the final state dict. Blocks the calling thread until the graph reaches END. Not suitable for long-running graphs in async web servers.
Scripts, tests, synchronous apps Blocks — use ainvoke() in async contexts
ainvoke()
await graph.ainvoke(input: dict, config: dict = None) -> dict
Async version of invoke(). Use inside async def endpoints. Required when your nodes contain await calls. Works with async checkpointers (AsyncPostgresSaver, AsyncSqliteSaver).
FastAPI, async apps, await in nodes Requires async checkpointer if you use one
stream()
for chunk in graph.stream(input, config, stream_mode='values'): ...
Streams state updates as the graph executes. Yields after each node completes. stream_mode controls what is yielded: full state values, per-node updates, or message chunks. Use for real-time progress reporting.
CLI tools, progress displays stream_mode='updates' is most efficient
astream()
async for chunk in graph.astream(input, config, stream_mode='values'): ...
Async version of stream(). Yields state updates without blocking. Combine with FastAPI's StreamingResponse for SSE endpoints. The correct approach for web applications.
FastAPI SSE, real-time UIs stream_mode='messages' only works with MessagesState
astream_events()
async for event in graph.astream_events(input, config, version='v2'): ...
Streams fine-grained lifecycle events: LLM start/stream/end, tool start/end, node start/end. The most detailed streaming API. Use version='v2' always. Filter events by event["event"] field.
Token-level streaming, debugging Always pass version='v2' — v1 is deprecated
get_state()
graph.get_state(config: dict) -> StateSnapshot
Returns the current (latest) state of a thread. StateSnapshot has .values (the state dict), .next (pending node names), .metadata, and .config. Requires a checkpointer. Essential for inspecting paused graphs.
Inspect paused/completed threads Returns None if thread_id has no checkpoints
get_state_history()
for snapshot in graph.get_state_history(config, limit=None): ...
Yields StateSnapshot objects for every checkpoint in a thread, newest first. Use for time-travel debugging, audit trails, and migration verification. limit controls how many snapshots to return.
Debugging, audit, migration First yielded is the latest checkpoint
update_state()
graph.update_state(config, values: dict, as_node: str = None)
Manually patch the state of a paused or completed thread. Creates a new checkpoint with the merged values. Use for: schema migrations, manual corrections, injecting external data. as_node controls which node is credited with the update.
HITL, schema migration, manual override Does not re-run any nodes — just writes state

State Design Patterns
TypedDict · Annotated reducers · MessagesState · Optional fields
▾
Basic Patterns
TypedDict with Defaults (Python 3.11+)
Use when: your state has optional fields or fields with sensible zero-values. Python 3.11+ supports Required/NotRequired. For older versions, use total=False and manage defaults manually.
from typing import TypedDict, NotRequired

class State(TypedDict):
    # Required fields
    query: str
    # Optional with default-like semantics
    result: NotRequired[str]          # absent from initial state
    iteration: NotRequired[int]       # default is 0 conceptually
    metadata: NotRequired[dict]       # default is {}

# In nodes, guard optional fields:
def my_node(state: State) -> dict:
    result = state.get("result", "")  # safe access
    count = state.get("iteration", 0)
    return {"iteration": count + 1}
Annotated Reducer — List Append (operator.add)
Use when: multiple parallel branches write to the same list field and you want to collect all results (not just the last write). Also correct for message history accumulation in custom state.
import operator
from typing import Annotated, TypedDict

class State(TypedDict):
    # Each parallel branch returns {"results": [item]}
    # operator.add concatenates all branch lists
    results: Annotated[list[str], operator.add]

    # For sets: merge without duplicates
    seen_ids: Annotated[set[str], lambda a, b: a | b]
Annotated Reducer — Custom Merge Logic
Use when: you need domain-specific merge behavior, such as keeping only unique entries, taking the max value, or merging dictionaries.
from typing import Annotated, TypedDict

def keep_latest(old: dict, new: dict) -> dict:
    """Merge two dicts, with new values taking precedence."""
    return {**old, **new}

def take_max(a: int, b: int) -> int:
    return max(a, b)

class State(TypedDict):
    metadata: Annotated[dict, keep_latest]
    high_score: Annotated[int, take_max]
MessagesState (Built-in)
Use when: your graph uses LangChain message objects (HumanMessage, AIMessage, ToolMessage). MessagesState has a built-in messages field with smart reducer that handles message deduplication by ID.
from langgraph.graph import MessagesState

# MessagesState is equivalent to:
# class MessagesState(TypedDict):
#     messages: Annotated[list[BaseMessage], add_messages]

class MyState(MessagesState):
    # Add your own fields on top
    topic: str
    summary: str

# add_messages reducer deduplicates by message ID
# and handles message updates correctly
Nested State
Use when: you have logical groupings of related fields (e.g., user profile, document metadata). Note: LangGraph's reducers apply at the top-level key only — nested dict merges require a custom reducer.
from typing import TypedDict, Annotated

class UserProfile(TypedDict):
    name: str
    language: str
    preferences: dict

class State(TypedDict):
    # Nested TypedDict as a field
    user: UserProfile
    query: str

    # With a custom reducer for safe nested merging:
    user_prefs: Annotated[dict, lambda old, new: {**old, **new}]

Node Design
Function signature · LLM calls · tool execution · error handling · async
▾
Core Rules

The Three Laws of Nodes

  1. Return only what changed. Return a dict with only the keys whose values you modified. LangGraph merges your return dict into the existing state — returning the full state object causes reducer bugs.
  2. Do not mutate the state object. The state parameter is a snapshot. Never write state["key"] = value. Return {"key": value} instead.
  3. Nodes are pure-ish functions. Side effects (API calls, DB writes) are fine, but the return value must be deterministic given the same state. Avoid global mutable state inside nodes.
Standard Node Signature
from typing import Any
from langchain_core.runnables import RunnableConfig

def my_node(state: State, config: RunnableConfig = None) -> dict:
    """
    state: current state snapshot (read-only conceptually)
    config: runtime config dict with thread_id, tags, metadata, etc.
    Returns: dict with only the keys you want to update
    """
    # Access config values
    thread_id = config["configurable"].get("thread_id") if config else None
    user_id   = config["configurable"].get("user_id") if config else None

    # Read from state
    current_count = state.get("count", 0)

    # Do work
    new_count = current_count + 1

    # Return only changes — NOT the full state
    return {"count": new_count}
LLM Call in Node
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate

llm = ChatOpenAI(model="gpt-4o-mini", temperature=0)

prompt = ChatPromptTemplate.from_messages([
    ("system", "You are a helpful assistant. User name: {name}"),
    ("human", "{query}"),
])

def llm_node(state: State) -> dict:
    chain = prompt | llm
    response = chain.invoke({
        "name": state.get("user_name", "User"),
        "query": state["query"]
    })
    return {"response": response.content}

# Async version:
async def llm_node_async(state: State) -> dict:
    response = await (prompt | llm).ainvoke({...})
    return {"response": response.content}
Tool Execution in Node
from langchain_core.tools import tool

@tool
def search_database(query: str) -> str:
    """Search the product database for items matching query."""
    # Implementation here
    return f"Found: product A, product B for '{query}'"

def tool_node(state: State) -> dict:
    tool_call = state["messages"][-1].tool_calls[0]
    tool_name = tool_call["name"]
    tool_args = tool_call["args"]

    tools_map = {"search_database": search_database}
    result = tools_map[tool_name].invoke(tool_args)

    from langchain_core.messages import ToolMessage
    return {"messages": [ToolMessage(
        content=result,
        tool_call_id=tool_call["id"]
    )]}
Error Handling in Node
import logging
logger = logging.getLogger(__name__)

def resilient_node(state: State) -> dict:
    try:
        result = call_external_api(state["query"])
        return {"result": result, "error": None}
    except TimeoutError as e:
        logger.warning(f"API timeout in resilient_node: {e}")
        return {"result": None, "error": f"timeout: {e}", "retry_count": state.get("retry_count", 0) + 1}
    except Exception as e:
        logger.error(f"Unexpected error in resilient_node: {e}", exc_info=True)
        # Return a safe default — do not let the graph crash
        return {"result": "unavailable", "error": str(e)}

Edge Patterns
add_edge · add_conditional_edges · Command routing · START/END constants
▾
Unconditional Edge
Use when: a node always proceeds to the same next node, no routing logic needed.
from langgraph.graph import START, END

builder.add_edge(START, "first_node")       # entry point
builder.add_edge("node_a", "node_b")        # a always goes to b
builder.add_edge("last_node", END)           # exit point

# Common mistake: omitting the edge to END
# Results in: NodeInterrupt or the graph running forever
# Always ensure every terminal node has an edge to END
Conditional Edge with Dict Mapping
Use when: a node can route to multiple possible next nodes based on state. The router function returns a string key; the dict maps keys to node names.
def route_decision(state: State) -> str:
    """Router function: receives state, returns a routing key string."""
    if state["score"] >= 70:
        return "pass"
    elif state["retry_count"] < 3:
        return "retry"
    else:
        return "fail"

builder.add_conditional_edges(
    "evaluator",         # source node
    route_decision,      # router function
    {                    # mapping: router return -> node name
        "pass":  "finalize",
        "retry": "evaluator",  # loop back
        "fail":  END,
    }
)

# If the router returns the node name directly (not a key),
# omit the mapping dict:
builder.add_conditional_edges("evaluator", route_decision)
# route_decision must return actual node names or END
Routing to Multiple Nodes (Fan-Out)
Use when: you want to dispatch to multiple nodes simultaneously from one conditional edge. Return a list of Send objects.
from langgraph.types import Send

def fan_out_router(state: State):
    """Return a list of Send objects to dispatch multiple nodes in parallel."""
    branches = state["branches"]  # e.g., ["historical", "technical", "recent"]
    return [
        Send("worker_node", {"branch": b, "topic": state["topic"]})
        for b in branches
    ]

builder.add_conditional_edges(
    START,
    fan_out_router,
    ["worker_node"]   # list of possible destination node names
)
Command-Based Routing (from inside a node)
Use when: a node needs to decide its own successor AND optionally update state simultaneously. Command is the modern replacement for conditional edges in many patterns. Also used to resume interrupted graphs.
from langgraph.types import Command
from typing import Literal

def smart_node(state: State) -> Command[Literal["node_a", "node_b", "__end__"]]:
    # Do work
    result = process(state)

    # Return Command to route AND update state atomically
    if result.quality > 0.8:
        return Command(
            goto="finalize",
            update={"result": result.content, "quality": result.quality}
        )
    elif state.get("retry_count", 0) < 3:
        return Command(
            goto="smart_node",   # loop to self
            update={"retry_count": state.get("retry_count", 0) + 1}
        )
    else:
        return Command(goto="__end__")

# Resume an interrupted graph:
graph.invoke(Command(resume={"decision": "approve"}), config=config)

Checkpointer Reference
MemorySaver · SqliteSaver · PostgresSaver · RedisSaver · Async variants
▾
Checkpointer Environment Import Path Initialization Limitations
MemorySaver Dev Only langgraph.checkpoint.memory MemorySaver() Lost on process exit. Single-process only. Never use in production.
SqliteSaver Dev / Single-server Prod langgraph.checkpoint.sqlite SqliteSaver.from_conn_string(":memory:") or "my.db" Single-process writes. Cannot be shared across application instances. File locking issues under high concurrency.
AsyncSqliteSaver Dev / Async apps langgraph.checkpoint.aiosqlite async with AsyncSqliteSaver.from_conn_string("my.db") as cp: Same limitations as SqliteSaver. Async interface only.
PostgresSaver Production (single/multi-server) langgraph.checkpoint.postgres PostgresSaver.from_conn_string(CONN_STR) + .setup() Requires psycopg v3. Must call setup() once to create tables. Connection not thread-safe — create per-request.
AsyncPostgresSaver Production (async) langgraph.checkpoint.postgres.aio async with AsyncPostgresSaver.from_conn_string(CONN_STR) as cp: Requires psycopg v3 async. Best for FastAPI/async apps. Use connection pools (psycopg.AsyncConnectionPool) under load.
RedisSaver High-Throughput / Distributed langgraph.checkpoint.redis (3rd party) RedisSaver.from_conn_string(REDIS_URL) TTL-based expiry can cause data loss. No built-in history. Best for ephemeral short-lived threads at high scale.

Human-in-the-Loop Patterns
interrupt_before · interrupt_after · Command(resume) · timeout handling
▾
interrupt_before vs. interrupt_after
# interrupt_before: pause BEFORE the named node executes
# Use when: you want human to review/approve before the action occurs
graph = builder.compile(
    checkpointer=memory,
    interrupt_before=["send_email", "execute_trade"]
)

# interrupt_after: pause AFTER the named node executes
# Use when: you want human to review output before proceeding
graph = builder.compile(
    checkpointer=memory,
    interrupt_after=["draft_response", "generate_code"]
)

# Check what is pending when paused:
snapshot = graph.get_state(config)
print(snapshot.next)    # tuple of pending node names
print(snapshot.values)  # current state at pause point
interrupt() Inside a Node (Modern Pattern)
from langgraph.types import interrupt

def review_node(state: State) -> dict:
    # Pause and collect human input
    # The argument to interrupt() is sent to the caller
    human_input = interrupt({
        "message": "Please review and respond:",
        "context": state["draft"],
        "options": ["approve", "revise", "reject"]
    })
    # Execution resumes here after Command(resume=...) is called
    return {"human_decision": human_input["choice"]}
Resuming an Interrupted Graph
from langgraph.types import Command

# --- First call: runs until interrupt ---
result = graph.invoke(initial_input, config=config)
# Graph is now paused. result may be partial.

# --- Inspect paused state ---
state = graph.get_state(config)
print("Paused at:", state.next)
print("Draft:", state.values.get("draft"))

# --- Resume with human decision ---
final = graph.invoke(
    Command(resume={"choice": "approve"}),
    config=config  # SAME config / thread_id
)

# --- Resume and jump to a different node ---
# (e.g., skip send and go straight to archive)
graph.invoke(
    Command(resume={"choice": "skip"}, goto="archive_node"),
    config=config
)
Handling Interrupt Timeouts
LangGraph does not have a built-in timeout for human approval. Implement your own using a background scheduler or a separate process that monitors idle threads.
from datetime import datetime, timezone, timedelta
import asyncio

async def timeout_monitor(graph, config: dict, timeout_hours: int = 24):
    """Background task: auto-reject approvals idle beyond timeout."""
    await asyncio.sleep(timeout_hours * 3600)
    state = graph.get_state(config)
    if state and state.next:  # still paused
        print(f"[TIMEOUT] Thread {config['configurable']['thread_id']} timed out")
        # Auto-reject
        graph.invoke(
            Command(resume={"choice": "timeout_rejected"}),
            config=config
        )

Multi-Agent Patterns
Supervisor · Handoff · Subgraph embedding · State schema design
▾
Supervisor Pattern
One orchestrator LLM decides which specialist to call next. The supervisor routes based on task requirements. Specialists return results to the supervisor for integration. Best for tasks where only one specialist is needed at a time (sequential, not parallel).
import operator
from typing import Annotated, Literal, TypedDict
from langchain_openai import ChatOpenAI
from langgraph.types import Command

SPECIALISTS = ["researcher", "writer", "coder"]

class SupervisorState(TypedDict):
    task: str
    completed_work: Annotated[list[str], operator.add]
    next_agent: str
    final_answer: str

def supervisor(state: SupervisorState) -> Command:
    llm = ChatOpenAI(model="gpt-4o-mini")
    prompt = f"""Task: {state['task']}
Work done so far: {state['completed_work']}
Available specialists: {SPECIALISTS} or FINISH.
Which specialist should act next? Reply with just the name."""
    decision = llm.invoke(prompt).content.strip().lower()
    if "finish" in decision:
        return Command(goto="finalize", update={"next_agent": "finalize"})
    agent = next((a for a in SPECIALISTS if a in decision), SPECIALISTS[0])
    return Command(goto=agent, update={"next_agent": agent})

def researcher(state: SupervisorState) -> Command:
    result = f"[Research] Found information about: {state['task']}"
    return Command(goto="supervisor", update={"completed_work": [result]})
Handoff Pattern (Command-based)
An agent that completes its task hands off directly to the next agent using Command(goto=). No central supervisor. Better for linear pipelines where each agent knows its successor. Less flexible than supervisor for dynamic routing.
from langgraph.types import Command

def research_agent(state) -> Command:
    findings = do_research(state["topic"])
    # Hand off directly to writer — no supervisor needed
    return Command(
        goto="writing_agent",
        update={"research_findings": findings}
    )

def writing_agent(state) -> Command:
    draft = write_from_findings(state["research_findings"])
    return Command(
        goto="review_agent",
        update={"draft": draft}
    )
Subgraph as a Node
Compile a graph independently, then embed it as a node in a parent graph. The subgraph's state schema must be compatible with (a subset of) the parent state. The subgraph sees only its own state keys.
from langgraph.graph import StateGraph, START, END

# Define and compile the subgraph independently
sub_builder = StateGraph(SubState)
sub_builder.add_node("step1", step1_fn)
sub_builder.add_edge(START, "step1")
sub_builder.add_edge("step1", END)
sub_graph = sub_builder.compile()  # independently testable

# Embed in parent graph
parent_builder = StateGraph(ParentState)
parent_builder.add_node("ingest", sub_graph)   # compiled graph as node
parent_builder.add_node("process", process_fn)
parent_builder.add_edge(START, "ingest")
parent_builder.add_edge("ingest", "process")
parent_builder.add_edge("process", END)
parent_graph = parent_builder.compile()

Streaming Reference
stream_mode options · astream_events types · FastAPI SSE · WebSocket
▾
stream() and astream() Modes
'values'
Yields the complete state dict after every node completes. Highest data volume. Use when you need the full state at each step for display or logging. Default mode.
'updates'
Yields a dict of only the changes each node made. Format: {"node_name": {"changed_key": value}}. Most efficient. Best for incremental UI updates.
'messages'
Yields individual message tokens as they stream from LLM nodes. Only works with MessagesState. Ideal for chat UIs showing token-by-token output.
'custom'
Yields data you emit manually via StreamWriter inside nodes. Gives you full control over what gets streamed and when. For specialized streaming requirements.
astream_events() Event Types
on_chain_start Fires when any LangChain runnable (including graph nodes) begins. event["name"] is the node name. Use to announce "Entering node: X" in your UI.
on_chain_end Fires when a runnable completes. event["data"]["output"] contains the return value. Use to log completion or update a progress bar.
on_chat_model_start Fires when an LLM call begins. Contains the messages being sent. Useful for logging prompts sent to the model.
on_chat_model_stream Fires for each token the LLM streams. event["data"]["chunk"].content is the token string. The primary event for token-by-token UI streaming.
on_chat_model_end Fires when LLM call completes. Contains token usage, finish reason. Use for cost tracking per node.
on_tool_start Fires when a tool is invoked. event["name"] is the tool name. event["data"]["input"] is the arguments. Use to show "Using tool: X" in UI.
on_tool_end Fires when a tool call completes. event["data"]["output"] is the tool result. Use to log tool call latency and output.
FastAPI SSE Pattern
from fastapi import FastAPI
from fastapi.responses import StreamingResponse
import json

app = FastAPI()

async def event_generator(message: str, thread_id: str):
    config = {"configurable": {"thread_id": thread_id}}
    async for event in graph.astream_events(
        {"messages": [("human", message)]},
        config=config,
        version="v2"
    ):
        if event["event"] == "on_chat_model_stream":
            token = event["data"]["chunk"].content
            if token:
                yield f"event: token\ndata: {json.dumps({'text': token})}\n\n"
        elif event["event"] == "on_chain_start":
            yield f"event: node\ndata: {json.dumps({'name': event['name']})}\n\n"
    yield f"event: done\ndata: {json.dumps({'status': 'ok'})}\n\n"

@app.post("/stream")
async def stream_endpoint(body: dict):
    return StreamingResponse(
        event_generator(body["message"], body.get("thread_id", "default")),
        media_type="text/event-stream",
        headers={"Cache-Control": "no-cache", "X-Accel-Buffering": "no"}
    )

Common Error Messages and Fixes
The 10 most frequent errors with exact messages and solutions
▾
GraphRecursionError: Recursion limit of 25 reached
Your graph has a loop that runs more than 25 times without reaching END. Usually a router function with a logic bug, or a loop termination condition that never triggers.
Fix: Increase limit: graph.compile(recursion_limit=50) or pass in config. More importantly, audit your router function. Add logging to see which node is looping. Check your termination condition — does it actually reach the exit branch?
ValueError: Node 'my_node' is not reachable from START
You called add_node("my_node", fn) but forgot to add an edge leading to it. Unreachable nodes cause compile() to fail with this error.
Fix: Add the missing edge: builder.add_edge("previous_node", "my_node") or remove the orphan node if you no longer need it.
KeyError: 'thread_id' not found in configurable
You compiled with a checkpointer (MemorySaver, PostgresSaver, etc.) but invoked without passing a thread_id in the config dict.
Fix: Always pass config={"configurable": {"thread_id": "some-id"}} when using any checkpointer. Every invoke call on a checkpointed graph needs a thread_id.
InvalidUpdateError: At key 'messages': Received update [AIMessage(...)] for field that does not have a reducer
You have a messages field in state that uses plain list type (no reducer) but parallel branches are both writing to it, causing a conflict.
Fix: Use Annotated[list, operator.add] or use MessagesState which has the add_messages reducer built in. Never use a plain list for fields written by multiple parallel nodes.
ValueError: Edge from '__start__' to 'my_node' already exists
You called add_edge(START, "my_node") twice, or you have both add_edge(START, ...) and set_entry_point(...) (the old API).
Fix: Call add_edge(START, "my_node") exactly once. Remove any set_entry_point() calls — it is deprecated. Use only the START constant from langgraph.graph.
TypeError: Node function must accept a single positional argument (state)
Your node function signature is wrong — it has no arguments, wrong argument names, or uses *args/**kwargs incorrectly.
Fix: Node signature must be def my_node(state: State) -> dict or def my_node(state: State, config: RunnableConfig) -> dict. No other signatures are accepted.
langgraph.errors.NodeInterrupt raised at node 'X'
The graph hit an interrupt() call inside a node (or interrupt_before/interrupt_after node). This is expected behavior when using HITL — the graph is paused, not broken.
Fix: This is not an error — it means the graph is waiting. Call graph.invoke(Command(resume=value), config=config) to resume. If you see it unexpectedly, check that your checkpointer config is correct.
psycopg.OperationalError: connection to server at "localhost" failed
Your PostgreSQL instance is not running, or the connection string is wrong, when using PostgresSaver.
Fix: Start Postgres (docker run -e POSTGRES_PASSWORD=dev -p 5432:5432 postgres:16). Verify connection string. Check firewall. Call PostgresSaver.setup() once to create the required tables before first use.
ImportError: cannot import name 'Send' from 'langgraph.graph'
You are importing Send from the wrong module, or your LangGraph version is too old.
Fix: Import from langgraph.types: from langgraph.types import Send, Command, interrupt. Upgrade to LangGraph >= 0.2.0 if you are on an older version.
ValueError: Missing required field 'X' in state update returned by node 'Y'
Your state schema has a required field (no default, not NotRequired) that is absent from the initial input dict passed to invoke().
Fix: Ensure your initial graph.invoke({"field1": val, "field2": val, ...}) includes all required TypedDict fields. Or mark optional fields as NotRequired[T] in your state schema.

Performance Configuration
recursion_limit · max_concurrency · async setup · timeout patterns
▾
ParameterDefaultDescription and When to Change
recursion_limit 25 Maximum node executions before GraphRecursionError. Counts every node visit, not just loops. For graphs with many sequential nodes, increase this. For agentic loops, set to 50-100. Never set to infinity in production.
max_concurrency None (unlimited) Maximum number of parallel node executions for Send-based fan-out. Set this when parallel branches make external API calls with rate limits. Example: graph.compile(max_concurrency=5) limits parallel branches to 5 concurrent.
debug False Enable verbose logging of every node execution and state transition. graph.compile(debug=True). Never enable in production — logs sensitive state data and adds significant overhead.
checkpointer None Attaching a checkpointer adds serialization overhead per node (~5-20ms for SQLite, ~15-40ms for Postgres depending on state size). Profile with and without for latency-sensitive applications.
Async Checkpointing Setup (FastAPI)
from contextlib import asynccontextmanager
from langgraph.checkpoint.postgres.aio import AsyncPostgresSaver
import psycopg_pool

CONN_STRING = "postgresql://postgres:dev@localhost:5432/langgraph"

# Application-level connection pool
pool = psycopg_pool.AsyncConnectionPool(
    CONN_STRING,
    min_size=2,
    max_size=10,
    open=False
)

@asynccontextmanager
async def lifespan(app):
    await pool.open()
    async with pool.connection() as conn:
        checkpointer = AsyncPostgresSaver(conn)
        await checkpointer.setup()
    app.state.pool = pool
    yield
    await pool.close()

app = FastAPI(lifespan=lifespan)

@app.post("/chat")
async def chat(body: dict, request: Request):
    async with request.app.state.pool.connection() as conn:
        checkpointer = AsyncPostgresSaver(conn)
        graph = builder.compile(checkpointer=checkpointer)
        result = await graph.ainvoke(body["input"], config=body["config"])
    return result

import asyncio

async def node_with_timeout(state: State) -> dict:
    """Async node with a hard timeout for external API calls."""
    try:
        result = await asyncio.wait_for(
            call_slow_external_api(state["query"]),
            timeout=30.0  # 30 second hard limit
        )

What LangGraph Studio Shows

Graph Topology

Real-time visualization of your graph's nodes and edges as your code runs. Highlights the currently executing node. Updates automatically on hot reload.

State Inspector

See the exact state dict at each node execution. Click through checkpoints to see state at any point in the run. Essential for debugging unexpected state values.

Thread History

Browse all threads and their checkpoint history. Time-travel: re-run from any checkpoint by clicking it. Explore how state evolved across invocations.

HITL Interrupt UI

When a graph pauses on an interrupt, Studio shows the current state and provides a UI to type your approval or revision. Test human-in-the-loop flows without building a custom UI first.
