Patterns & Anti-patterns

Best Practices and Common Pitfalls in LangChain Development

✅ Pattern: LCEL Composition

Use the pipe operator for chain composition

chain = prompt | llm | parser
✅ Clear, composable, streaming support

❌ Anti-pattern: Chain Nesting

Don't nest LLMChains deeply

# Bad
chain = LLMChain(llm=LLMChain(llm=llm, prompt=p1), prompt=p2)
❌ Hard to debug, no streaming

✅ Pattern: RunnablePassthrough

Pass through unchanged data

chain = {
  "context": retriever | format_docs,
  "question": RunnablePassthrough()
} | prompt | llm
✅ Clean data flow through chain

❌ Anti-pattern: No Input Validation

Don't trust user input directly

# Bad
chain.invoke(user_input) # SQL injection risk
❌ Security vulnerabilities

✅ Pattern: Fallback Chains

Always have fallback models

chain = primary_llm.with_fallbacks([fallback_llm])
✅ Graceful degradation

❌ Anti-pattern: Ignoring Token Limits

Don't send unlimited context

# Bad - will fail on long docs
chain.invoke(large_document)
❌ API errors, truncation

✅ Pattern: Structured Outputs

Use Pydantic for validation

class Response(BaseModel):
  answer: str
  confidence: float

parser = PydanticOutputParser(pydantic_object=Response)
✅ Type-safe, auto-validation

❌ Anti-pattern: String Concatenation

Don't build prompts manually

# Bad
prompt = "Question: " + question + " Context: " + context
❌ Brittle, error-prone

✅ Pattern: Semantic Caching

Cache similar queries

@lru_cache(maxsize=1000)
def get_embedding(text: str):
  return embeddings.embed_query(text)
✅ 10x faster for repeated queries

❌ Anti-pattern: Synchronous Calls

Don't block on async operations

# Bad in async context
result = chain.invoke(input) # Blocks
❌ Poor performance

✅ Pattern: Stream Responses

Stream tokens for better UX

for chunk in chain.stream(input):
  print(chunk.content, end="", flush=True)
✅ 5x faster perceived speed

❌ Anti-pattern: No Error Handling

Don't let exceptions propagate

# Bad
result = chain.invoke(input) # Can crash
❌ Poor user experience

Key Principles

  • Use LCEL for composition
  • Always validate inputs/outputs
  • Implement fallbacks
  • Use async for I/O
  • Stream responses
  • Monitor with LangSmith
  • Write tests for chains
  • Document prompt decisions

Patterns evolve - stay current with the community
