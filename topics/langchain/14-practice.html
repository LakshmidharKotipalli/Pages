Section 14: Practice Exercises

Learn by Building

Theory without practice is just trivia. These exercises progress from beginner-friendly chains to production-grade systems. Each exercise includes the problem statement, architecture blueprint, key decisions, and success criteria. Build all of them and you'll have a portfolio that proves LangChain mastery.

Exercise 1: Simple Chain — Haiku Generator (Beginner)

Objective

Build your first LCEL chain that generates haikus about any given topic. Introduces the fundamental chain pattern: Prompt → Model → Parser.

Time estimate: 30 minutes

Architecture

  • Input: Topic string from user
  • Prompt template: Instructs LLM to write a 5-7-5 syllable haiku
  • Model: GPT-3.5 Turbo (temperature 0.9 for creativity)
  • Output parser: String output parser (clean text)
  • Chain: Prompt | Model | Parser using LCEL pipe operator

Success Criteria

  • Chain produces valid haiku output
  • Works with at least 5 different topics
  • Uses LCEL pipe operator (not legacy LLMChain)
  • Bonus: Add streaming output
  • Bonus: Generate haikus in different languages

Exercise 2: RAG System — PDF Question Answering (Intermediate)

Objective

Build a complete RAG pipeline that answers questions about any PDF document. This is the most commonly asked LangChain interview exercise.

Time estimate: 2-3 hours

Requirements

  • Load a PDF document
  • Split text into optimal chunks (500 chars, 100 overlap)
  • Create embeddings and store in vector database
  • Retrieve relevant chunks with MMR search
  • Generate answers grounded in retrieved context
  • Return source page citations with every answer

Architecture Blueprint

PDF RAG Pipeline

Success Criteria

  • Can load and query any PDF successfully
  • Answers include source page citations
  • Returns "I don't know" for questions not in the document
  • Bonus: Add re-ranking for better retrieval quality
  • Bonus: Support multiple PDFs with metadata filtering
  • Bonus: Add a simple Streamlit UI

Exercise 3: Tool-Using Agent (Intermediate)

Objective

Build an agent that dynamically chooses between multiple tools to answer complex questions requiring reasoning and external data.

Time estimate: 2-3 hours

Architecture

  • 3+ Custom Tools: Search, Calculator, Date/Time
  • Agent Type: OpenAI Functions Agent (reliable tool selection)
  • Safety Limits: max_iterations=10, timeout=60s
  • Error Handling: Graceful parse error recovery
  • Logging: All reasoning steps captured

Agent Decision Flow

ReAct Pattern: Reason → Act → Observe → Repeat

Success Criteria

  • Agent correctly selects the right tool for different query types
  • Handles multi-step queries requiring multiple tools
  • Gracefully handles tool errors without crashing
  • Bonus: Add web search tool (Tavily/SerpAPI)
  • Bonus: Add conversation memory so agent remembers context

Exercise 4: Conversational Chatbot with Memory (Intermediate)

Objective

Build a multi-turn chatbot that remembers context across messages and supports configurable personas.

Time estimate: 1-2 hours

Architecture

  • Memory Type: Summary Buffer Memory (auto-summarizes old messages)
  • Token Limit: 1,000 tokens max in memory
  • Personas: Configurable system prompts (helpful, pirate, teacher, debug)
  • Chat Template: System message + memory history + human input
  • Reset Capability: Clear memory to start fresh conversation

Success Criteria

  • Bot remembers user's name and previous context across turns
  • Memory stays within token limits (doesn't grow unbounded)
  • Different personas produce noticeably different responses
  • Bonus: Persist memory to Redis for session persistence
  • Bonus: Build a Gradio or Streamlit chat interface

Exercise 5: Multi-Agent Research Team (Advanced)

Objective

Build a multi-agent system where specialized agents collaborate to produce a comprehensive research report.

Time estimate: 4-6 hours

Agent Roles

  • Researcher Agent: Gathers information using search tools
  • Writer Agent: Produces clear, structured content from research
  • Critic Agent: Reviews for accuracy and completeness
  • Coordinator: Orchestrates the workflow between agents

Multi-Agent Pipeline Architecture

Coordinator → Researcher → Writer → Critic → Revision → Final Output

Success Criteria

  • Pipeline produces a polished article from just a topic string
  • Critic provides substantive feedback that improves final output
  • Each agent has a clear, distinct role visible in output
  • Bonus: Add web search to the researcher for real data
  • Bonus: Implement with LangGraph for stateful graph execution

Exercise 6: Production RAG with Evaluation (Advanced)

Objective

Build a production-quality RAG pipeline with comprehensive evaluation, monitoring, and optimization. The capstone project.

Time estimate: 6-8 hours

Architecture Components

  • Multi-source ingestion: PDF, web, markdown document loaders
  • Hybrid search: Dense embeddings (60%) + sparse BM25 (40%) + re-ranking
  • Semantic caching: Redis-based cache for similar queries
  • LangSmith evaluation: Gold-standard dataset with QA, helpfulness, relevance metrics
  • Streaming responses: Server-sent events with source citations
  • Cost tracking: Per-query cost with model routing
  • Automated testing: Regression tests against evaluation datasets

Success Criteria

  • Achieves >85% accuracy on evaluation dataset
  • Implements caching with measurable cost savings
  • Full LangSmith tracing with searchable metadata
  • Streaming responses under 500ms time-to-first-token
  • Bonus: A/B test different chunk sizes and measure impact
  • Bonus: Deploy as API using LangServe
