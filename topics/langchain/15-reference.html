Section 15: Quick Reference

LangChain Cheat Sheet

The essential architecture patterns, component choices, and configuration decisions you'll reach for every day. Bookmark this page for fast lookup during development and interviews.

LCEL Architecture Patterns

Simple Chain

Pattern: Prompt → LLM → Output Parser

The most basic pattern. Takes a template with variables, sends to LLM, parses the response as clean text. Use for single-step generation tasks.

RAG Chain

Pattern: {context: Retriever | Formatter, question: Passthrough} → Prompt → LLM → Parser

Retrieves relevant documents, formats them into context, injects into prompt alongside the question. The workhorse of document QA.

Parallel Execution

Pattern: RunnableParallel(summary=chain_a, keywords=chain_b, sentiment=chain_c)

Runs multiple chains simultaneously on the same input. Great for extracting multiple properties in parallel — 3x faster than sequential.

Conditional Routing

Pattern: RunnableBranch((condition_A → chain_A), (condition_B → chain_B), default_chain)

Routes input to different chains based on conditions. Use for multi-model routing ("simple" → GPT-3.5, "complex" → GPT-4).

Fallback Chain

Pattern: primary_llm.with_fallbacks([fallback_llm])

If GPT-4 fails (rate limited, timeout), automatically falls back to GPT-3.5 or Claude. Essential for production reliability.

Streaming Output

Pattern: chain.stream(input) → yields token chunks as they arrive

5x better perceived speed. User sees tokens appearing immediately instead of waiting for the full response. Free with LCEL.

Agent Architecture

Agent Setup Checklist

  1. Define tools with clear descriptions (what it does + when to use it)
  2. Choose agent type: OpenAI Functions Agent (recommended) or ReAct
  3. Create prompt with system message + human input + agent scratchpad
  4. Create agent executor with safety limits
  5. Set max_iterations=10, max_execution_time=60
  6. Enable handle_parsing_errors=True
  7. Enable return_intermediate_steps=True for debugging
  8. Enable verbose=True during development

Agent Safety Rules

  • Always set max_iterations (5-10 for most use cases)
  • Always set max_execution_time (30-120 seconds)
  • Always enable parse error handling
  • Limit tools to 5-7 per agent (more = worse selection)
  • Validate tool inputs before execution
  • Rate limit expensive or external tool calls
  • Monitor agent step count in real-time

Retrieval Enhancement Stack

  1. Base retriever: Vector similarity search (k=5-10)
  2. + BM25: Add keyword search for hybrid (40/60 split)
  3. + Re-ranking: Cross-encoder re-orders by true relevance
  4. + Threshold: Filter out docs below 0.76 similarity
  5. Result: Top 3 highly relevant documents to LLM

Debugging Quick Reference

Quick Diagnostic Steps

  1. Enable verbose mode to see intermediate steps
  2. Enable LangSmith tracing (just set env vars)
  3. Test each component in isolation
  4. Compare a working trace with the failing one
  5. Check the prompt first (80% of issues)
  6. Set temperature=0 for reproducible debugging

Common Errors → Fixes

Version Pinning (Always Pin!)

Always pin exact versions in requirements.txt. LangChain releases frequently with breaking changes. Example: langchain==0.1.17, langchain-openai==0.0.8, langchain-community==0.0.24, langchain-core==0.1.52
