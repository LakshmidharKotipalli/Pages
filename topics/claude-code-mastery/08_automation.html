Automation & CI/CD

Run Claude headlessly in scripts, CI/CD pipelines, and batch operations at scale.

Headless Mode

claude -p — Non-Interactive Execution

Process prompts without an interactive session. Perfect for scripting and automation.

# One-off queries claude -p "Explain what this project does" # Structured JSON output for scripts claude -p "List all API endpoints" --output-format json # Streaming for real-time processing claude -p "Analyze this log file" --output-format stream-json # Plain text output (default) claude -p "Summarize recent changes" --output-format text
Batch Operations

Fan Out Across Files

Generate a Task List

list all 2,000 Python files that need migrating

Write a Script to Loop

for file in $(cat files.txt); do claude -p "Migrate $file from React to Vue. Return OK or FAIL." \ --allowedTools "Edit,Bash(git commit *)" done

Test on a Few, Then Scale

Use --allowedTools to scope permissions. Use --verbose for debugging during testing phase.

CI/CD Integration

GitHub Actions & GitLab

🐙

GitHub Actions

An official Claude Code GitHub Action brings AI automation to your workflows:

  • Auto-review PRs for bugs, style, and security
  • Generate test scaffolds for changed files
  • Implement features from issue descriptions
  • Respond to @claude mentions in PRs and issues
  • Create detailed changelogs from commits

Store your Anthropic API key as a repository secret. Trigger workflows on pull_request events or @claude mentions.

🦊

GitLab CI/CD

Integrate Claude Code into GitLab pipelines for AI-powered merge request assistance:

  • Auto-propose merge requests from issue descriptions
  • Respond to @claude in MR comments
  • Run automated code review on diffs
  • Generate documentation for changed code

Store the Anthropic API key as a masked, protected CI/CD variable. Use event-driven triggers for MR and issue comments.

Unix Utility

Pipe, Chain & Integrate

Use Claude as a standard Unix utility in your existing command-line workflows.

# Monitor logs for anomalies tail -f app.log | claude -p "Slack me if you see any anomalies" # Automate translations in CI claude -p "translate new strings into French and raise a PR for review" # Security review changed files git diff main --name-only | claude -p "review these changed files for security issues" # Chain with other commands claude -p "your prompt" --output-format json | jq '.result' | your_command # Code review as verification step claude -p "Review @src/auth.ts for injection vulnerabilities. Output 'PASS' or 'FAIL: reason'"
🔒 Safe Autonomous Mode

For fully autonomous operation, --dangerously-skip-permissions bypasses all prompts. Only use inside sandboxed environments (Docker containers, CI runners). Prefer /sandbox for OS-level isolation that restricts filesystem and network access while allowing freedom within those boundaries.
