Extending Claude Code

Skills, Hooks, MCP Servers, Custom Subagents, and Plugins — unlock Claude's full potential with the extensibility system.

Extension System

Five Ways to Extend

🎯 Skills — Reusable Slash Commands

Skills are SKILL.md files in .claude/skills/ that define reusable workflows invoked via /slash-commands. They're the most accessible way to extend Claude Code.

Example: Fix GitHub Issues

--- name: fix-issue description: Fix a GitHub issue disable-model-invocation: true --- Analyze and fix the GitHub issue: $ARGUMENTS. 1. Use `gh issue view` to get the issue details 2. Understand the problem described in the issue 3. Search the codebase for relevant files 4. Implement the necessary changes to fix the issue 5. Write and run tests to verify the fix 6. Ensure code passes linting and type checking 7. Create a descriptive commit message 8. Push and create a PR

Usage: /fix-issue 1234

Example: API Conventions

--- name: api-conventions description: REST API design conventions for our services --- # API Conventions - Use kebab-case for URL paths - Use camelCase for JSON properties - Always include pagination for list endpoints - Version APIs in the URL path (/v1/, /v2/)
Key Concepts

• Skills are loaded on-demand to minimize token usage
• Use $ARGUMENTS to accept arguments from the slash command
• Set disable-model-invocation: true for deterministic scripts that don't need AI reasoning
• Can include scripts/, examples/, and resources/ subdirectories

⚡ Hooks — Automated Event Triggers

Shell scripts that run automatically when specific events occur during a session. Configure via /hooks command or directly in .claude/settings.json.

Hook Events

Event When It Fires Use Case
PreToolUse Before Claude executes a tool Validation, security blocks
PostToolUse After tool execution completes Auto-formatting, logging
UserPromptSubmit Before Claude processes your prompt Inject dynamic context
Notification When Claude needs your input Desktop alerts, sounds

Desktop Notification Hook (macOS)

osascript -e 'display notification "Claude Code needs your attention" with title "Claude Code"'

Security: Hooks can block dangerous commands or modifications to sensitive files. They provide a deterministic control layer over Claude's behavior.

🔌 MCP Servers — External System Connections

The Model Context Protocol connects Claude to external systems — databases, APIs, monitoring tools, issue trackers, documentation sites.

claude mcp add

What MCP Enables

  • Query your production database directly
  • Fetch Sentry error details and stack traces
  • Read and search Notion/Confluence documentation
  • Interact with Jira/Linear issues
  • Browse documentation sites
  • Access any API through custom MCP servers

MCP is an open-source standard — any tool can expose an MCP interface for Claude to use.

🤖 Custom Subagents — Specialized AI Workers

Create purpose-specific agents in .claude/agents/. Each subagent has its own system prompt, tool access, and can even use different models (e.g., Opus for deep review).

--- name: security-reviewer description: Reviews code for security vulnerabilities tools: Read, Grep, Glob, Bash model: opus --- You are a senior security engineer. Review code for: - Injection vulnerabilities (SQL, XSS, command injection) - Authentication and authorization flaws - Secrets or credentials in code - Insecure data handling Provide specific line references and suggested fixes.

Key Benefits

  • Isolated context: Subagent work doesn't pollute your main conversation
  • Specialized prompts: Each agent focuses on one expertise area
  • Tool restriction: Limit what each agent can access for safety
  • Model selection: Use Opus for deep analysis, Sonnet for speed
🧩 Plugins — Community Extensions

Install community-built plugins with /plugin. Plugins bundle skills, subagents, hooks, and resources into shareable packages.

  • Code Intelligence Plugin: Adds precise "go to definition" and "find references" navigation for your programming language
  • Frontend Design Plugin: Enhances UI/UX code generation with design system knowledge
  • Browse and discover plugins from the community marketplace
Decision Guide

Which Extension to Use?

Need Use Why
Reusable workflow Skill Slash command invocation, filesystem-based
Auto-run on events Hook Deterministic, no AI reasoning needed
External data access MCP Server Standard protocol for tool connection
Specialized AI task Subagent Isolated context, custom system prompt
Shareable package Plugin Bundles multiple extensions together
