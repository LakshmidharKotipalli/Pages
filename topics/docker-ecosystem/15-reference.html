📄 Dockerfile Instructions Reference ▼
── FOUNDATION ──────────────────────────────────────── FROM node:18-alpine # Base image. First instruction always. FROM node:18-alpine AS builder # Named stage for multi-stage builds ── METADATA ────────────────────────────────────────── LABEL maintainer="team@company.com" # Key-value metadata LABEL version="1.0" description="API" ── WORKING DIRECTORY ───────────────────────────────── WORKDIR /app # Set working dir (creates if missing) ── ENVIRONMENT ─────────────────────────────────────── ENV NODE_ENV=production # Available at build AND run time ARG BUILD_VERSION # Only at build time: --build-arg ── COPYING FILES ───────────────────────────────────── COPY package*.json ./ # Copy files into image layer COPY --chown=node:node . . # Copy with ownership (run as node) COPY --from=builder /app/dist ./dist # Multi-stage: copy from another stage ADD https://example.com/file . # ADD also handles URLs and .tar.gz ── RUNNING COMMANDS ────────────────────────────────── RUN apt-get update && apt-get install -y curl && rm -rf /var/lib/apt/lists/* RUN --mount=type=cache,target=/root/.npm npm ci # BuildKit cache mount RUN --mount=type=secret,id=token cat /run/secrets/token # BuildKit secret ── SECURITY ────────────────────────────────────────── RUN addgroup -S app && adduser -S app -G app # Alpine: create non-root user USER app # Switch to non-root (must come late) ── PORTS ───────────────────────────────────────────── EXPOSE 8080 # Documentation only, doesn't publish ── VOLUMES ─────────────────────────────────────────── VOLUME ["/data"] # Declare mount point (auto volume if not mounted) ── HEALTH CHECK ────────────────────────────────────── HEALTHCHECK --interval=30s --timeout=5s --retries=3 \ CMD wget -qO- http://localhost:8080/health || exit 1 ── ENTRY POINT ─────────────────────────────────────── ENTRYPOINT ["node"] # Exec form: receives signals correctly CMD ["server.js"] # Default args to ENTRYPOINT, or default command # ENTRYPOINT + CMD: docker run myapp index.js → runs node index.js # CMD alone: docker run myapp alt.js → runs node alt.js (overrides CMD)

⚡ Performance Tuning ▼

Build Performance

  • Enable BuildKit: export DOCKER_BUILDKIT=1 (default in Docker 23+)
  • Order layers: COPY slow-changing files first, fast-changing last
  • Use --mount=type=cache for package managers (npm, pip, apt)
  • Use registry cache in CI: --cache-from and --cache-to type=registry
  • Minimize build context with .dockerignore

Runtime Performance

  • Set CPU limits based on profiled peak, not guesses
  • JVM: -XX:MaxRAMPercentage=75 respects container memory limits
  • Use host networking (--network host) only when network overhead is measurable
  • Use tmpfs for high-write-frequency temporary files
  • Tune log driver: json-file with max-size and max-file to prevent log bloat

Storage Performance

  • OverlayFS is the default and best for most cases
  • For database containers: use named volumes, not bind mounts (better performance)
  • Enable buildkit BuildCache GC to auto-prune old build cache
