Large Language Models
The computational engines that compress the internet's knowledge into billions of weighted parameters, then reconstruct it one token at a time.
Open Full Guide →
Architecture: Transformer-based neural networks using self-attention (Q×Kᵀ/√d → V) to let every token attend to every other token in parallel — solving RNN's information bottleneck.
Training Pipeline: Pre-training (next-token prediction on massive corpora) → SFT (supervised fine-tuning on instruction pairs) → RLHF (human preference alignment via reward model + PPO).
Scaling Laws: Kaplan et al. (2020) showed loss decreases as a smooth power law with more params/data/compute — making improvements predictable and justifying billion-dollar investments.
Key Terms: BPE (tokenizer), KV Cache (GPU memory bottleneck during inference), MoE (Mixture of Experts — 47B params, 13B active), RoPE (rotary position embeddings for context extension).
Evolution: 2017 Transformer paper → GPT-2 (2019) → GPT-3 few-shot (2020) → ChatGPT + RLHF mainstream (2022) → Reasoning models o1/o3/DeepSeek-R1 (2024-2026).
Production Concerns: Hallucination mitigation, "lost in the middle" problem, quantization (INT4 = 1-3% accuracy loss), speculative decoding (2-3x speedup), prompt injection defense, cost-optimized serving pipelines.
At Scale: Semantic caching, model routing (small→large based on complexity), continuous batching (vLLM/TGI), multi-model evaluation frameworks, and LoRA fine-tuning (0.1% of parameters).
Analogy: "The Library of Compressed Thought" — LLMs compress humanity's written knowledge into weighted parameters and decompress plausible continuations on demand.

AI Agents, Tools, Skills & MCP
The architecture of intelligent systems that reason, act, and compose — connected by a universal protocol.
Open Full Guide →
Agent Definition: A reasoning engine that perceives → plans → acts → observes in a loop, pursuing goals across multiple turns — NOT just a smarter chatbot. Goal-directed behavior with real-world consequences.
Tools: Discrete, callable capabilities (read_file, web_search, query_db) with strict schemas. Keep tools atomic and single-purpose — a tool that does too much breaks reasoning chains.
Skills: Proven, named multi-step workflows (search → read → summarize). Reduce token waste, encode institutional knowledge, become building blocks for complex behavior — like clinical protocols in medicine.
MCP (Model Context Protocol): Anthropic's open standard (Nov 2024) — the "USB-C of AI connectivity." Uses JSON-RPC 2.0 over stdio or SSE. Enables capability discovery: agents work with new tools without redeployment.
Key Patterns: ReAct loop (Reason→Act→Observe), Orchestrator pattern (coordinator + specialist sub-agents), Human-in-the-Loop gates, Tool Composition, Graceful Degradation tiers.
Anti-Patterns: God Agent (50+ tools), Tool Sprawl (>15 tools/agent), Prompt-as-Architecture (compensating for missing structure), Implicit State (relying on conversation history vs explicit state).
Production Reality: Context window management, retry storm prevention (exponential backoff + circuit breakers), MCP gateway design (auth, rate limiting, versioning), observability with OpenTelemetry spans.
Analogy: Hospital system — agents are doctors pursuing patient outcomes, tools are medical instruments, skills are clinical protocols, MCP is the standardized equipment interface.

LangChain
The orchestration layer that transforms raw LLM capabilities into production-ready applications through composable chains, memory, and tools.
Open Full Guide →
Core Purpose: Solves LLM statelessness, boilerplate prompt management, provider switching, and tool access. Python & JavaScript libraries for building LLM applications.
LCEL: LangChain Expression Language — declarative pipe syntax: chain = prompt | model | parser. Built-in streaming, async, batch, and parallel execution via Runnables.
Ecosystem: LangSmith (tracing, evals, monitoring), LangServe (one-line REST API deployment on FastAPI), LangChain Templates (RAG, agents, chatbot reference architectures).
Key Components: PromptTemplates (reusable, testable), ChatModels (provider-agnostic), OutputParsers (structured extraction), Document Loaders/Splitters, Vector Store integrations, Memory modules.
RAG Pattern: Document loading → text splitting → embedding → vector store → retrieval → generation. Production RAG adds re-ranking, hybrid search, metadata filtering, and evaluation.
Patterns: Chain composition, retrieval-augmented generation, conversational agents with memory, multi-modal pipelines. Anti-patterns: over-abstraction, ignoring LangSmith tracing, monolithic chains.
Trade-offs: Performance vs complexity, cost vs capability. Decision frameworks for when to use LangChain vs raw API calls vs alternatives (LlamaIndex, Haystack).
Analogy: Restaurant kitchen — the chef (LLM) needs prep cooks (tools), recipe cards (prompts), a kitchen manager (chain), and quality control (output parsers) to serve great meals reliably.

LangGraph
The stateful, graph-based framework that transforms AI agents from unpredictable scripts into reliable, production-grade workflows.
Open Full Guide →
Central Insight: An AI agent is not a function call — it is a process. Processes have state, can pause/resume, fail partway and be retried from checkpoints, and need observability.
Core Primitives: StateGraph (the graph), Nodes (processing units), Edges (transitions — conditional or unconditional), State (typed data flowing through), Checkpointer (persistence).
Checkpointers: MemorySaver (dev), PostgresSaver (production), RedisSaver (high-throughput). Enable time-travel debugging, fault recovery, and human-in-the-loop patterns.
Key Patterns: ReAct loop, supervisor multi-agent, hierarchical routing, parallel fan-out/fan-in, approval gates. Anti-patterns: God nodes, stateless graphs, missing cycle breakers.
Human-in-the-Loop: Interrupt nodes pause execution at approval gates. Graph state is persisted, human reviews, then execution resumes. Built-in with interrupt_before/interrupt_after.
Ecosystem: LangSmith (tracing), LangGraph Platform (managed deployment), LangGraph Studio (visual IDE/debugger), LangChain Core (components), LangServe (API exposure).
At Scale: Checkpointer bottlenecks at 1M invocations, thread ID design, state explosion management, concurrent graph execution, horizontal scaling, cost management strategies.
Analogy: Air Traffic Control — StateGraph is the airspace, nodes are control towers, edges are flight paths, state is the airplane + cargo, checkpointer is the black box flight recorder.

Docker & Ecosystem
From first container to fleet orchestration — containerization that solved "works on my machine" and changed software delivery forever.
Open Full Guide →
What It Is: OS-level virtualization using Linux namespaces (isolation) and cgroups (resource limits). Not a VM — containers share the host kernel, making them lightweight and fast to start.
Architecture: Docker Client → Docker Daemon → containerd → runc. Images are layered (Union filesystem), built from Dockerfiles, and stored in registries (Docker Hub, ECR, GCR).
Key Concepts: Images (immutable templates), Containers (running instances), Volumes (persistent data), Networks (bridge, host, overlay), Docker Compose (multi-container definitions).
Dockerfile Best Practices: Multi-stage builds (reduce image size), order instructions for cache efficiency, use .dockerignore, don't run as root, pin base image versions, minimize layers.
Trade-offs: VMs vs Containers (isolation vs speed), Docker vs Podman (daemon vs daemonless), Compose vs Swarm vs K8s (single-host vs simple orchestration vs full orchestration).
At Scale Challenges: 1000+ containers → registry bottlenecks, image bloat, log aggregation nightmares. Solutions: distroless images, layer caching strategies, container-native logging (Fluentd/Loki).
Ecosystem: Podman (daemonless alternative), containerd (industry-standard runtime), Kata Containers (VM-level isolation), Buildah (OCI image builder), Skopeo (image inspection).
Evolution: 2013 Docker launch → 2015 container revolution → 2017 Docker vs K8s (K8s wins orchestration) → 2019 K8s deprecates Docker runtime → Present: containerd standard.

Kubernetes & Ecosystem
A data center operating system that treats entire fleets of servers as a single computer — from Google's Borg to industry standard.
Open Full Guide →
Core Insight: Declarative configuration + self-healing controllers = reliable infrastructure at scale. Tell K8s WHAT you want (desired state), not HOW to do it. Controllers continuously reconcile actual→desired.
Architecture: Control Plane (API Server, Scheduler, Controller Manager, etcd) + Worker Nodes (kubelet, kube-proxy, container runtime). Control Plane is "air traffic control."
Key Primitives: Pod (smallest deployable unit), Deployment (declarative updates), Service (stable network endpoint), Ingress (external traffic routing), ConfigMap/Secret, PersistentVolume.
Patterns: Sidecar containers, init containers, DaemonSets, StatefulSets, Jobs/CronJobs, Horizontal Pod Autoscaler. Anti-patterns: no resource limits, latest tag, exposed control planes.
Ecosystem: Helm (package manager), Istio/Linkerd (service mesh), Prometheus/Grafana (monitoring), ArgoCD/Flux (GitOps), Ingress Controllers (NGINX, Traefik).
When NOT to Use: Simple apps (overkill), single server (don't bother), no expertise (complex failure modes). K8s solves orchestration but introduces distributed systems complexity.
At Scale (10K+ nodes): etcd performance limits, scheduler latency, network policy explosion, OOMKilled epidemics, RBAC complexity. Production reality: monitoring, logging, security, GitOps are mandatory.
History: Google Borg (2003) → Docker (2013) → K8s open-sourced (2014) → CNCF (2016) → Docker Swarm defeated (2017) → K8s becomes "Linux of cloud" (2020+). From Borg to industry standard.

Vision Language Models
The sensory organs of AI — translating the spatial, physical world into the discrete language of reasoning.
Open Full Guide →
Why VLMs Exist: Pure text models hallucinate physics, geometry, and spatial context they cannot "see." VLMs emerged because language is a low-bandwidth compression of reality — models must parse pixels directly.
Architecture: Image Encoder (ViT — treats images as patch sequences) → Projection Layer (aligns visual features to language space) → LLM Decoder (generates text from fused representations).
Key Terms: ViT (Vision Transformer), CLIP (contrastive image-text pre-training), Cross-Attention (language model "looks back" at visual features), AnyRes (dynamic resolution patching).
Major Models: CLIP (bridge model), LLaVA (open-source pioneer), Qwen-VL (layout-aware), GPT-4V (frontier multimodal), Gemini (natively multimodal), PaLI (Google's VLM family).
Applications: Document understanding (OCR + reasoning), visual question answering, image captioning, GUI navigation, medical imaging analysis, autonomous driving perception.
Trade-offs: Resolution vs compute (higher resolution = quadratic token growth), accuracy vs latency, general vs specialized models, open-source vs API-based deployment.
At Scale Challenges: Image preprocessing pipelines, token budget management (images consume thousands of tokens), multi-image reasoning, video understanding frame sampling strategies.
Evolution: CLIP (2021) → Flamingo (2022) → LLaVA (2023) → GPT-4V/Gemini multimodal (2023-2024) → Layout-aware models Qwen-VL (2024-2025) → Real-time video understanding.

Claude Code Ecosystem
An agentic terminal-based AI development environment — essentially an operating system for AI-assisted software engineering.
Open Full Guide →
What It Is: A lead AI agent that orchestrates specialized sub-agents, persistent memory, extensible skills, and external tools through layered context management for terminal-native AI-assisted development.
Context Window: The 200K token context window governs what Claude can "see." Context management (/clear, /compact, /rewind) is the hidden superpower separating power users from beginners.
Agent Architecture: Main agent (director) → sub-agents (specialists). Like a film production studio: director orchestrates cinematographers, editors, sound engineers following a production bible (CLAUDE.md).
CLAUDE.md: Persistent project context file — the AI's memory. Hierarchical locations (global, project, directory) define coding standards, project architecture, and behavioral rules.
Extensibility: Skills (reusable workflows in markdown), Hooks (pre/post action triggers), MCP Servers (external tool integration), Custom Subagents, Plugins — full extension ecosystem.
Multi-Agent: Git worktrees for parallel sessions, agent teams via /teleport, cross-device workflows. Each agent works independently but contributes to a single coherent project.
Competitors: GitHub Copilot (IDE-integrated), Cursor (fork of VS Code), Cody (Sourcegraph), Windsurf — each with different strengths in code generation, context handling, and workflow.
At Scale: 50+ developer teams, enterprise rollouts, massive codebases. Challenges: auto-memory management, token cost optimization, consistent style enforcement, security boundaries.

Claude Code Mastery
The definitive guide to transforming your development workflow — from Anthropic's official docs, engineers who built it, and battle-tested community patterns.
Open Full Guide →
Golden Workflow: Explore → Plan → Implement → Commit. The four-phase approach Anthropic engineers use internally. Always start with exploration before writing code.
CLAUDE.md Deep Dive: The most critical config file. Three hierarchy levels: ~/.claude/CLAUDE.md (global), project root (project), subdirectory (directory-specific). Include: coding standards, architecture, gotchas.
Prompting Mastery: Context injection via @ references, piping data (cat file | claude), image analysis, the "interview technique" for complex features — ask Claude to propose a plan before implementing.
Context Management: /clear (reset), /compact (summarize history), /rewind (undo recent turns), subagents for isolated tasks. These commands are essential for long development sessions.
Extending Claude Code: Skills (markdown-defined workflows), Hooks (pre/post-action scripts), MCP Servers (Postgres, GitHub, filesystem integration), Custom Subagents, Plugins marketplace.
Automation & CI/CD: Headless mode (claude -p "task" --allowedTools), batch operations, GitHub Actions integration, GitLab CI/CD pipelines, Unix piping workflows for automated code review.
Anti-Patterns: (1) Vague prompts, (2) Never compacting (context pollution), (3) Ignoring CLAUDE.md (reinventing conventions), (4) Not reviewing diffs (trusting blindly), (5) One massive session (context decay).
Key Stats: 50% average productivity gain, 59% work usage at Anthropic, 30hr continuous task focus capability, 1M token context window. Community-proven tips: use git worktrees, limit scope, verify output.
