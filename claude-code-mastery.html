What Is Claude Code?

An agentic coding tool that reads your codebase, edits files, runs commands, and integrates with your development tools.

Core Concept

Not Autocomplete — An Agent

Claude Code doesn't just suggest completions. It thinks through problems, explores codebases, plans changes, executes solutions, runs tests, and iterates — all autonomously within a single conversation.

🖥️

Terminal-Native Power

Claude Code lives where developers live. It operates directly in your terminal with full access to your tools, filesystem, and shell — no context switching needed. It's built to be your pair programmer in the environment you already use.

🧠

Agentic Architecture

Powered by an agentic loop: Claude reads context, decides which tool to use, executes it, observes results, and repeats until the task is done. It can chain dozens of actions — reading files, running commands, editing code — all without manual intervention.

🔌

Available Everywhere

Use Claude Code in your Terminal, VS Code, Cursor, JetBrains IDEs, the Desktop App, on the Web (claude.ai/code), via iOS, through GitHub Actions, GitLab CI/CD, Slack, and even Chrome.

Capabilities

What You Can Do

Build Features & Fix Bugs

Describe what you want, and Claude explores the codebase, identifies the right files, implements changes, writes tests, and verifies everything works.

Automate Git Workflow

Create commits with descriptive messages, open pull requests, handle code reviews, and even respond to PR comments — all through natural language.

claude "commit my changes with a descriptive message"

Write & Run Tests

Find untested code, generate test scaffolding, add edge cases, run the suite, and fix any failures — all in one prompt.

claude "write tests for the auth module, run them, and fix any failures"

Connect External Tools (MCP)

Use the Model Context Protocol to connect Claude to databases, monitoring tools, issue trackers, documentation sites, and any API.

Run Agent Teams

Coordinate multiple Claude instances working in parallel — one writes code, another reviews, a third writes tests. Experimental agent teams with a lead coordinator.

Pipe, Script & Automate

Use Claude as a Unix utility: pipe in logs, diff outputs, or file lists. Process them headlessly and chain into your existing command-line workflows.

git diff main --name-only | claude -p "review these changed files for security issues"

The Golden Workflow

Explore → Plan → Implement → Commit. The four-phase approach Anthropic engineers use internally for maximum effectiveness.

The Framework

Four Phases of Effective Development

This is how Anthropic's own engineers work with Claude Code. Don't skip phases — each one builds on the previous.

1. Explore — Understand Before Changing

Ask Claude to read and understand the existing code before making any changes. Let it map dependencies, trace execution flows, and identify relevant patterns.

read /src/auth and understand how we handle sessions and login. also look at how we manage environment variables for secrets.

Point Claude to specific directories. Be precise about what you want it to understand. Don't just say "look at the project."

2. Plan — Think Before Acting

Use Plan Mode (Shift+Tab twice or Ctrl+G) to prevent Claude from writing code. Get a structured implementation plan and review it carefully.

I want to add Google OAuth. What files need to change? What's the session flow? Create a plan.
Why Plan Mode Matters

In Plan Mode, Claude can only read files and use the AskUserQuestion tool — it can't edit or run anything. This forces deep analysis without the risk of premature implementation. Review the plan, refine it, then let Claude execute.

3. Implement — Execute with Verification

After approving the plan, let Claude implement. Always tell it to write tests and verify its work. This is the single highest-leverage instruction you can give.

implement the OAuth flow from your plan. write tests for the callback handler, run the test suite and fix any failures.

4. Commit — Ship with Context

Let Claude create meaningful commits and PRs that include full context about what changed and why.

commit with a descriptive message and open a PR
Critical Principle

Verification Is Everything

✅ The #1 Best Practice

"The single highest-leverage thing you can do is give Claude a way to verify its own work."

Include tests, expected outputs, or screenshot comparisons. When Claude can check itself, performance improves dramatically. If you can't verify it, don't ship it.

— Anthropic Official Best Practices

Verification Strategies

Unit Tests

"Write tests, run them, and fix any failures" — the most reliable verification loop.

Type Checking

Tell Claude to run npm run typecheck or mypy after changes. Add this to your CLAUDE.md.

Linting

Run npm run lint or equivalent. Claude will fix issues automatically if instructed.

Screenshots

For frontend work, use the Chrome extension or paste images to let Claude compare visual output against designs.

CLAUDE.md — Your Project's AI Memory

The most critical configuration file for Claude Code. It gives Claude persistent context about your project across every session.

Why It Matters

The Impact of CLAUDE.md

❌ Without CLAUDE.md

Claude guesses your conventions. Uses wrong import styles. Doesn't know your test commands. Can't find your project structure. You repeat yourself every session. Wrong patterns. Wrong tools.

✅ With CLAUDE.md

Claude follows your patterns from the first prompt. Uses correct styles, tools, and commands. Understands your architecture. Carries context across sessions. Consistent quality.

Template

What to Include

# Code Style - Use ES modules (import/export), not CommonJS (require) - Destructure imports when possible - Use TypeScript strict mode - Prefer functional components over class components - Use kebab-case for file names # Workflow - Run `npm run typecheck` after making a series of code changes - Prefer running single tests, not the whole suite, for performance - Use `npm run lint:fix` before committing - Always run `npm test -- --related` for changed files # Architecture See @README.md for project overview and @package.json for available npm commands. # Key Patterns - API routes live in /src/api/routes/ - Database models use Prisma in /prisma/schema.prisma - Auth middleware is in /src/middleware/auth.ts - Error handling follows /src/utils/errors.ts patterns # Additional Instructions - Git workflow: @docs/git-instructions.md - API conventions: @docs/api-conventions.md - Personal overrides: @~/.claude/my-project-instructions.md # Compaction Rules When compacting, always preserve: - The full list of modified files - Any test commands used - Key architectural decisions made
Configuration

Hierarchical Locations

Claude reads CLAUDE.md files from multiple locations, with more specific ones taking precedence.

Location Scope Use Case
~/.claude/CLAUDE.md Global — all projects Personal preferences, universal conventions, preferred tools
./CLAUDE.md Project root Team conventions — check into git to share with team
./CLAUDE.local.md Project (personal) Your personal overrides — add to .gitignore
./src/CLAUDE.md Subdirectory Module-specific guidance — great for monorepos
../CLAUDE.md Parent directory Monorepo root conventions shared by sub-projects
💡 @ Imports

Use @path/to/file syntax to import additional instruction files. This keeps your main CLAUDE.md lean while referencing detailed docs:
See @README.md for project overview and @package.json for available npm commands.

⚠️ Common Mistake: Over-Specification

If your CLAUDE.md is too long, Claude ignores important rules lost in the noise. Ruthlessly prune. If Claude already does something correctly without the instruction, delete it. Convert repetitive rules to hooks instead. Keep it focused on what Claude can't infer on its own.

Advanced

Dynamic CLAUDE.md with Hooks

Use session hooks to inject dynamic context (recent commits, CI status, environment variables) into CLAUDE.md at session startup.

# In a UserPromptSubmit hook, inject recent git context: echo "Recent commits: $(git log --oneline -5)" >> /tmp/claude-context.md # Claude reads this dynamically each session

Prompting Like a Pro

Specific, rich, contextual prompts are the difference between average and extraordinary results.

Good vs Bad

Specificity Changes Everything

❌ Vague Prompt

"what would you improve in this file?"

No file specified. No scope. No criteria. Claude has to guess everything.

✅ Specific Prompt

"refactor the calculateTax function in @src/billing/tax.ts to handle EU VAT rules. Add unit tests covering zero-rate, standard-rate, and reverse-charge scenarios. Run the tests and fix failures."

Exact file, clear task, specific test cases, verification built in.

Techniques

Context Injection Methods

@

@ File References

Reference files with @ instead of describing where code lives. Claude reads the file contents before responding.

refactor @src/auth/oauth.ts following the patterns in @src/auth/saml.ts
|

Pipe In Data

Send file contents, logs, error outputs, and diffs directly into Claude's input.

# Pipe error logs cat error.log | claude "explain this error" # Pipe git diffs git diff main | claude -p "review this" # Pipe file lists find . -name "*.ts" | claude "which files handle auth?"
🖼️

Images

Copy/paste or drag-and-drop images directly into the prompt. Perfect for UI bugs, design mockups, and error screenshots.

[paste screenshot] Fix the layout to match this design. The button should be right-aligned.
🌐

URLs

Give URLs for documentation and API references. Use /permissions to allowlist frequently-used domains for automatic access.

🔍

Let Claude Fetch Context

Instead of providing everything upfront, tell Claude to pull context itself using Bash commands, MCP tools, or by reading files.

Find all files that import from the billing module and understand how they use the calculateTax function.
🎯

Codebase Questions

Ask Claude to understand code without changing it. Great for onboarding and knowledge transfer.

How does logging work? What does async move { ... } do on line 134? Why does this code call foo() instead of bar()?
Advanced Technique

Let Claude Interview You

For complex features where you haven't thought through all the details, let Claude ask the hard questions.

I want to build [brief description]. Interview me in detail using the AskUserQuestion tool. Ask about technical implementation, UI/UX, edge cases, concerns, and tradeoffs. Don't ask obvious questions — dig into the hard parts I might not have considered. Keep interviewing until we've covered everything, then write a complete spec to SPEC.md.
🧠 Why This Works

Developers often start coding before thinking through edge cases. The interview technique forces you to consider authentication flows, error handling, race conditions, and UX decisions before any code is written. The resulting spec becomes a better prompt for implementation.

Context Management

The hidden superpower that separates power users from beginners. Claude's performance degrades when its context fills with irrelevant information.

Core Strategies

Keep Context Clean & Focused

🧹

/clear — Reset Between Tasks

Use /clear between unrelated tasks. This fully resets the context window. Long sessions with mixed context significantly reduce Claude's performance.

# After finishing one task: /clear # Now start a completely new task
📦

/compact — Compress Context

When auto-compaction triggers, Claude summarizes what matters most. For more control, run it manually with focus instructions:

/compact Focus on the API changes /compact Keep the migration plan and all file paths /compact Preserve test commands and architectural decisions
⏪

Rewind — Checkpoint Restore

Press Esc + Esc or run /rewind to open the rewind menu:

  • Restore previous conversation and code state
  • Summarize from a selected message forward
  • Keep earlier context intact while compressing recent messages
🔍

Subagents for Investigation

Heavy research tasks can fill your main context. Offload them to subagents that search, explore, and report back without polluting your primary window.

use subagents to investigate how our auth system handles token refresh, and whether we have any existing OAuth utilities I should reuse
✋

Esc — Course-Correct Early

Press Esc to stop Claude mid-action. Context is preserved, so you can redirect immediately. Don't let Claude go far down the wrong path.

↩️

Resume Sessions

Resume previous work without losing context. Name sessions with /rename for easy retrieval.

claude --continue # Most recent claude --resume # Pick from list
Reference

Context Actions Quick Reference

Action How When to Use
Esc Stop mid-action, preserve context Claude is heading the wrong direction
Esc + Esc Open rewind/checkpoint menu Restore previous state or selective summarize
/clear Full context reset Between completely unrelated tasks
/compact [focus] Compress with instructions Context getting large, auto-compaction not enough
/rewind Menu to restore checkpoints Undo failed approaches or bad paths
"Undo that" Tell Claude to revert Quick rollback of recent code changes
--continue Resume latest session Continue where you left off
--resume Pick from sessions list Resume a specific named conversation
/rename [name] Name current session Create bookmarks for long-running work
🧠 Pro Tip: Compaction Rules in CLAUDE.md

Add instructions to your CLAUDE.md like: "When compacting, always preserve the full list of modified files and any test commands." This ensures critical context survives auto-summarization, so you never lose track of what's been changed.

📝 Handoff Documents

For long-running projects, before /clear, ask Claude: "Write a handoff document summarizing all work, decisions, remaining tasks, and files changed." Start the new session with that summary as input. Community-proven technique for preserving context across fresh sessions.

Extending Claude Code

Skills, Hooks, MCP Servers, Custom Subagents, and Plugins — unlock Claude's full potential with the extensibility system.

Extension System

Five Ways to Extend

🎯 Skills — Reusable Slash Commands

Skills are SKILL.md files in .claude/skills/ that define reusable workflows invoked via /slash-commands. They're the most accessible way to extend Claude Code.

Example: Fix GitHub Issues

--- name: fix-issue description: Fix a GitHub issue disable-model-invocation: true --- Analyze and fix the GitHub issue: $ARGUMENTS. 1. Use `gh issue view` to get the issue details 2. Understand the problem described in the issue 3. Search the codebase for relevant files 4. Implement the necessary changes to fix the issue 5. Write and run tests to verify the fix 6. Ensure code passes linting and type checking 7. Create a descriptive commit message 8. Push and create a PR

Usage: /fix-issue 1234

Example: API Conventions

--- name: api-conventions description: REST API design conventions for our services --- # API Conventions - Use kebab-case for URL paths - Use camelCase for JSON properties - Always include pagination for list endpoints - Version APIs in the URL path (/v1/, /v2/)
Key Concepts

• Skills are loaded on-demand to minimize token usage
• Use $ARGUMENTS to accept arguments from the slash command
• Set disable-model-invocation: true for deterministic scripts that don't need AI reasoning
• Can include scripts/, examples/, and resources/ subdirectories

⚡ Hooks — Automated Event Triggers

Shell scripts that run automatically when specific events occur during a session. Configure via /hooks command or directly in .claude/settings.json.

Hook Events

Event When It Fires Use Case
PreToolUse Before Claude executes a tool Validation, security blocks
PostToolUse After tool execution completes Auto-formatting, logging
UserPromptSubmit Before Claude processes your prompt Inject dynamic context
Notification When Claude needs your input Desktop alerts, sounds

Desktop Notification Hook (macOS)

osascript -e 'display notification "Claude Code needs your attention" with title "Claude Code"'

Security: Hooks can block dangerous commands or modifications to sensitive files. They provide a deterministic control layer over Claude's behavior.

🔌 MCP Servers — External System Connections

The Model Context Protocol connects Claude to external systems — databases, APIs, monitoring tools, issue trackers, documentation sites.

claude mcp add

What MCP Enables

  • Query your production database directly
  • Fetch Sentry error details and stack traces
  • Read and search Notion/Confluence documentation
  • Interact with Jira/Linear issues
  • Browse documentation sites
  • Access any API through custom MCP servers

MCP is an open-source standard — any tool can expose an MCP interface for Claude to use.

🤖 Custom Subagents — Specialized AI Workers

Create purpose-specific agents in .claude/agents/. Each subagent has its own system prompt, tool access, and can even use different models (e.g., Opus for deep review).

--- name: security-reviewer description: Reviews code for security vulnerabilities tools: Read, Grep, Glob, Bash model: opus --- You are a senior security engineer. Review code for: - Injection vulnerabilities (SQL, XSS, command injection) - Authentication and authorization flaws - Secrets or credentials in code - Insecure data handling Provide specific line references and suggested fixes.

Key Benefits

  • Isolated context: Subagent work doesn't pollute your main conversation
  • Specialized prompts: Each agent focuses on one expertise area
  • Tool restriction: Limit what each agent can access for safety
  • Model selection: Use Opus for deep analysis, Sonnet for speed
🧩 Plugins — Community Extensions

Install community-built plugins with /plugin. Plugins bundle skills, subagents, hooks, and resources into shareable packages.

  • Code Intelligence Plugin: Adds precise "go to definition" and "find references" navigation for your programming language
  • Frontend Design Plugin: Enhances UI/UX code generation with design system knowledge
  • Browse and discover plugins from the community marketplace
Decision Guide

Which Extension to Use?

Need Use Why
Reusable workflow Skill Slash command invocation, filesystem-based
Auto-run on events Hook Deterministic, no AI reasoning needed
External data access MCP Server Standard protocol for tool connection
Specialized AI task Subagent Isolated context, custom system prompt
Shareable package Plugin Bundles multiple extensions together

Automation & CI/CD

Run Claude headlessly in scripts, CI/CD pipelines, and batch operations at scale.

Headless Mode

claude -p — Non-Interactive Execution

Process prompts without an interactive session. Perfect for scripting and automation.

# One-off queries claude -p "Explain what this project does" # Structured JSON output for scripts claude -p "List all API endpoints" --output-format json # Streaming for real-time processing claude -p "Analyze this log file" --output-format stream-json # Plain text output (default) claude -p "Summarize recent changes" --output-format text
Batch Operations

Fan Out Across Files

Generate a Task List

list all 2,000 Python files that need migrating

Write a Script to Loop

for file in $(cat files.txt); do claude -p "Migrate $file from React to Vue. Return OK or FAIL." \ --allowedTools "Edit,Bash(git commit *)" done

Test on a Few, Then Scale

Use --allowedTools to scope permissions. Use --verbose for debugging during testing phase.

Multi-Agent Development

Run multiple Claude sessions simultaneously. Worktrees, agent teams, parallel sessions, and cross-device workflows.

Parallel Sessions

Git Worktrees — Isolated Parallel Work

Each Claude session gets its own isolated git worktree — no merge conflicts between parallel agents working on the same project.

# Start Claude in a named worktree # Creates .claude/worktrees/feature-auth/ with a new branch claude --worktree feature-auth # Start another session in a separate worktree claude --worktree bugfix-123 # Auto-generate a worktree name claude --worktree # Creates something like "bright-running-fox"

How It Works

  • Each worktree lives in <repo>/.claude/worktrees/<name>
  • A new branch worktree-<name> is created automatically
  • Full filesystem isolation — no conflicts between sessions
  • Cleanup available for completed worktrees

Use Cases

  • Work on a feature while a bug fix runs in parallel
  • Test Opus vs Sonnet on the same problem simultaneously
  • Run code review in one session while implementing in another
  • Explore alternative implementation strategies concurrently

Anti-Patterns & Pitfalls

Five common failure patterns that derail productivity — and how to fix them. Directly from Anthropic's official best practices.

Failure Patterns

Avoid These Traps

🍳

The Kitchen Sink Session

What happens: You start with one task, ask Claude something unrelated, then go back to the first task. The context fills with irrelevant information, and Claude's performance degrades.

✅ Fix

Use /clear between unrelated tasks. Every single time. If you need to context-switch, clear first.

🔄

Correcting Over and Over

What happens: Claude does something wrong, you correct it, it's still wrong, you correct again. The context becomes polluted with failed approaches, making each attempt worse.

✅ Fix

After two failed corrections, stop. Run /clear and write a better initial prompt that incorporates what you learned from the failures. A fresh start with a better prompt beats repeated corrections.

📜

The Over-Specified CLAUDE.md

What happens: Your CLAUDE.md grows to hundreds of lines. Important rules get buried in noise. Claude ignores critical instructions because they're lost in the volume.

✅ Fix

Ruthlessly prune. If Claude already does something correctly without the instruction, delete it. Convert repetitive patterns to hooks. Keep CLAUDE.md focused on what Claude can't infer on its own.

👁️

The Trust-Then-Verify Gap

What happens: Claude produces a plausible-looking implementation that doesn't handle edge cases. You trust it because it looks right, but it breaks in production.

✅ Fix

Always provide verification. Tell Claude to write tests, run them, and fix failures. If you can't verify the output (no tests, no type checking, no screenshots), don't ship it.

🌊

The Infinite Exploration

What happens: You ask Claude to "investigate" or "look into" something without scoping it. Claude reads hundreds of files, filling the context window with exploration data.

✅ Fix

Scope investigations narrowly: "Look at /src/auth to understand token refresh" (not "investigate our auth system"). For broad research, use subagents so exploration doesn't consume your main context.

Develop Your Intuition

When to Intervene

Experience is learning when to let Claude run versus when to course-correct. Here are signals to watch for.

🟢 Let Claude Continue When...

  • It's following your plan methodically
  • It's running tests and fixing failures iteratively
  • The file changes align with your expectations
  • It's asking clarifying questions instead of guessing

🔴 Intervene When...

  • It's reading dozens of unrelated files
  • It's rewriting something that already works
  • It's ignoring your architectural constraints
  • The same error has been corrected twice
  • It's generating boilerplate you didn't ask for

Mindset

How the Best Developers Use Claude Code

🎯

Treat It Like a Fast Junior Engineer

Claude is incredibly capable but needs clear direction. Don't assume it knows your context. Provide specific instructions, give it custom tools and commands, and course-correct early — before it goes far down the wrong path.

⚡

Auto-Accept for Prototyping

Use Shift+Tab to enable auto-accept mode for rapid prototyping. Give Claude abstract problems and let it generate 80% solutions autonomously. Then refine the last 20% manually. This is how Anthropic engineers prototype internally.

📝

Write Handoff Documents

Before clearing context on a long project, ask Claude to write a handoff document summarizing all work, decisions, remaining tasks, and files changed. Start your next session with this summary. This bridges the gap between sessions.

🧪

Test-Driven with Claude

Two powerful patterns: (1) Write tests first, then implement to pass them. (2) Run two sessions — one writes tests, another writes code. Both build verification directly into the development flow.

🔧

Give Claude Your CLI Tools

Claude excels when it can use your existing tools. Make sure gh, aws, gcloud, sentry-cli, kubectl are installed. Tell Claude: "Use 'foo-tool --help' to learn about the tool, then use it to solve X."

📊

Monitor Token Usage

Use /insights to analyze your coding patterns and identify friction points. Tools like ccusage track token costs. More thinking = higher costs, so be strategic about when to use extended thinking.

Multipliers

Productivity Multipliers

Install the VS Code Extension

Claude reads IDE diagnostics (type errors, syntax issues) in real-time, enabling continuous self-correction as it codes. This dramatically reduces the edit-check-fix cycle.

Name Your Sessions

Use /rename "oauth-migration" so you can easily find and resume complex multi-step work. Sessions named "debugging-memory-leak" are instantly findable with --resume.

Set Up Desktop Notifications

Configure /hooks → Notification to ping you when Claude needs input. Switch to other work while Claude crunches through a long task, and come back when needed.

Use Extended Thinking

For complex architectural decisions, enable thinking mode. Use keywords like "ultrathink" or "think deeply" before your prompt. Claude reasons more thoroughly about tradeoffs before acting.

Create Project-Specific Skills

Automate your most common workflows: deploy scripts, PR templates, migration patterns, test scaffolding — all as /slash-commands. Build once, reuse forever.

Use Git Bisect for Bug Hunting

Tell Claude to run git bisect to find the exact commit that introduced a bug. It handles the binary search and testing automatically.

Commit Frequently

Commit before letting Claude make big changes. This gives you an instant rollback point if things go wrong. Git commits are your safety net for autonomous coding.
